Parcours Développeur

1. Implémentation du SDK

L'intégration du SDK LaunchDarkly chez Desjardins suit des standards de sécurité stricts. Assurez-vous d'utiliser les clés d'environnement appropriées (Client-side vs Server-side).

Note : Cette section doit être complétée avec des exemples de code spécifiques (Java/TS/C#) utilisés chez Desjardins.

2. SOC2 et PR Automatiques

Toute création ou modification de flag doit passer par un processus d'approbation. Chez Desjardins, une PR automatique est générée pour chaque changement de configuration, nécessitant une approbation par une tierce personne.

3. Être "Toggle Ready"

Un code Toggle Ready est un code qui peut être désactivé instantanément sans impact sur la stabilité du système. Évitez les dépendances circulaires entre les flags.

Simulation Interactive : Sandbox de Toggle

Expérimentez comment le code réagit dynamiquement au changement d'un flag.

if (ldClient.variation("nouveau-calcul-interet", user)) {
    console.log("Exécution de la nouvelle logique...");
} else {
    console.log("Logique standard active.");
}
> Logique standard active.
