La pratique des Feature Toggles

Comprendre ce qu'est un feature toggle, ce qu'il n'est pas, et distinguer ses grands types pour l'utiliser à bon escient.

Définition

Un Feature Toggle (aussi appelé feature flag, switch ou interrupteur de fonctionnalité) est un mécanisme qui permet de modifier le comportement d'une application sans modifier ni redéployer son code. Concrètement, le code contient une condition :

if (client.estActif("nouvelle-page-virements")) {
    afficherNouvellePage();   // nouveau comportement
} else {
    afficherAnciennePage();   // comportement existant (sûr)
}

La valeur du toggle est évaluée au moment de l'exécution, à partir d'un service de configuration central (ici, LaunchDarkly). On peut donc changer le comportement en direct, pour tout le monde ou pour une partie ciblée des utilisateurs.

L'idée clé : séparer le déploiement (le code est en production) de la mise en exposition (les utilisateurs voient la fonctionnalité). Ce sont désormais deux décisions distinctes, prises par des personnes potentiellement différentes et à des moments différents.

Les grands types de toggles

Tous les toggles ne se ressemblent pas. On les classe généralement selon leur durée de vie et leur dynamisme (à quelle fréquence leur valeur change). Cette distinction, popularisée par Martin Fowler, aide à décider comment les gérer.

Toggle de livraison (Release)

Permet de déployer du code incomplet ou nouveau de façon graduelle (canari, pourcentage, segments) puis de généraliser. Durée de vie courte : il doit être retiré une fois la fonctionnalité pleinement déployée et stable.

  • Idéal pour le déploiement progressif et le trunk-based development.
  • À retirer rapidement pour éviter la dette de toggle.

Toggle d'expérimentation

Sert à comparer plusieurs variantes (A/B, multivarié) pour mesurer leur impact sur des indicateurs d'affaires. Sa valeur est attribuée de façon cohérente par utilisateur le temps de l'expérience.

  • Durée de vie = durée de l'expérience.
  • Nécessite des métriques fiables pour conclure.

Toggle opérationnel (Ops)

Contrôle un aspect opérationnel du système : activer/désactiver une intégration coûteuse, dégrader gracieusement une fonctionnalité sous charge. Le fameux kill switch en fait partie.

  • Certains sont permanents (kill switch de sécurité).
  • Pilotés par les équipes d'exploitation / SRE.

Toggle de permission

Expose des fonctionnalités à des catégories d'utilisateurs précises (clients premium, employés, rôles). Durée de vie longue, voire permanente, car il porte une règle d'affaires durable.

  • À distinguer d'un vrai système de gestion des droits.
  • Change rarement de logique, mais reste actif.

Ce qu'un toggle n'est pas

Un toggle…n'est pas…
Un interrupteur d'exposition temporaireUn système de configuration permanent pour tout
Un outil de gestion du risque de déploiementUn substitut aux tests automatisés
Un moyen de cibler des audiencesUn système d'autorisation / de sécurité robuste
Du code temporaire à retirerUne branche `if` à laisser indéfiniment
