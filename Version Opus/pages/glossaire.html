Glossaire

Le vocabulaire essentiel des feature flags, dans le contexte de LaunchDarkly et de ExampleCo.

Feature Toggle / Feature Flag
Interrupteur logiciel permettant d'activer ou de désactiver une fonctionnalité sans redéployer le code. Synonymes : flag, switch, interrupteur de fonctionnalité.
Déploiement progressif (rollout)
Activation graduelle d'une fonctionnalité par paliers (ex. 1 % → 10 % → 50 % → 100 %) afin de limiter le rayon d'impact et de surveiller les indicateurs. Un premier petit palier est parfois appelé « canari ».
Kill switch
Capacité de désactiver instantanément une fonctionnalité problématique sans redéploiement, pour réduire le temps de réaction lors d'un incident.
Gating
Point du code où l'on évalue un flag pour bifurquer entre deux comportements. Un bon gating est centralisé, lisible et testable.
Valeur par défaut (fallback)
Valeur retournée par le SDK si le service de flags est injoignable. Elle doit correspondre au comportement existant et sûr.
Contexte (context)
Ensemble d'attributs (ex. id de membre, segment) fournis au SDK pour évaluer un flag de façon ciblée et reproductible.
Bucketing
Mécanisme qui répartit de façon déterministe les utilisateurs dans des « seaux » selon un hachage de leur clé, afin d'appliquer un pourcentage de rollout de manière cohérente.
Segment
Groupe d'utilisateurs défini par des critères (ex. bêta-testeurs, employés internes) que l'on peut cibler avec un flag.
Variation
Une des valeurs possibles d'un flag : ON/OFF pour un booléen, ou plusieurs valeurs pour un flag multivarié (chaîne, nombre, JSON).
Dette de toggle (stale flag)
Accumulation de toggles obsolètes jamais retirés, créant du code conditionnel mort qui complique tests et maintenance.
Trunk-based development
Pratique consistant à intégrer fréquemment sur une branche principale ; les feature toggles permettent de livrer du code incomplet sans l'exposer.
SOC2
Cadre de conformité. Chez ExampleCo, toute modification de flag crée automatiquement une Pull Request devant être approuvée par une personne différente de l'auteur (séparation des tâches).
Catalogue central
Référentiel ExampleCo recensant et classant l'ensemble des toggle switches : type, propriétaire, état, conformité.
Changement standard (ServiceNow)
Processus de gestion du changement utilisé pour les switches à risque modéré. Inadapté aux livraisons progressives, qui exigent un système sur mesure.
SDK
Bibliothèque intégrée à l'application qui interroge LaunchDarkly, met en cache les règles et évalue les flags pour un contexte donné.
Mise en production (MEP)
Étape de déploiement orchestré requise pour les switches de livraison progressive chez ExampleCo.

Aucun terme ne correspond à votre recherche.

Note : Ce glossaire doit être complété avec les termes et définitions propres au vocabulaire interne de ExampleCo (noms d'outils, rôles, acronymes spécifiques).
